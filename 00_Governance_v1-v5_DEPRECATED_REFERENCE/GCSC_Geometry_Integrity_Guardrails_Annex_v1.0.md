# 4.0 CORE INTEGRITY GUARDRAILS

Each guardrail is mandatory and non-negotiable.

---

## **G1 ‚Äî No Geometry in Governance**
Governance documents may not:
- Define numeric values  
- Suggest dimensions  
- Re-express geometry in text  
- Modify or restate parameters  

All geometry exists only in the Spec, Kernel, and Plates.

---

## **G2 ‚Äî Programmatic Plates Are Canonical**
Artistic drawings, AI images, or sketches:
- May illustrate  
- May clarify  
- May not override or imply geometry  

Only programmatic plates define the canonical shape.

---

## **G3 ‚Äî The Kernel Must Match the Spec 1:1**
The kernel:
- Must use Spec constants directly  
- Must not infer missing values  
- Must not ‚Äúsmooth,‚Äù ‚Äúadjust,‚Äù or ‚Äúinterpret‚Äù geometry  
- Must not use LLM-generated numbers  

Only deterministic math derived from the Spec is allowed.

---

## **G4 ‚Äî The Spec May Only Be Updated Through Governance**
Spec updates require:
- A governance discussion  
- New Spec version  
- Corresponding Kernel version bump  
- Regeneration of plates  

No direct edits.  
No assumptions.  
No LLM-generated ‚Äúfixes.‚Äù

---

## **G5 ‚Äî No Hidden Geometry Logic**
The kernel must not contain:
- Hidden offsets  
- Implicit constants  
- Non-linear adjustments unless defined in Spec  
- Developer-created ‚Äúcorrections‚Äù  

All math must map transparently to the Spec.

---

## **G6 ‚Äî Geometry Stability Over Time**
For a given Spec version:
- Output must be identical  
- Kernel must be deterministic  
- Plates must match bit-for-bit (aside from export metadata)  

If plates change unexpectedly ‚Üí Freeze-on-Failure.

## **G7 ‚Äî No Cross-Pillar Drift**
Governance may not:
- Describe geometry  
- Re-express geometry logic  
- Propose geometry values  

Geometry may not:
- Reference governance rules in code  
- Implicitly modify governance behavior  

Dual-Pillar separation is absolute.

## **G8 ‚Äî Strict Version Synchronization**
Valid geometry state requires all three aligned versions:

- Spec version  
- Kernel version  
- Plate version
